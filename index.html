A Simple Guide · 簡易說明

Deploy Your Website to GitHub Pages 把你的網站部署到 GitHub Pages

You received a folder of website files. Follow these six steps to put your site online — for free, in about 10 minutes. No coding required. 你收到一個網站資料夾。照著下面六個步驟，大約 10 分鐘就能把網站免費發布上線，完全不需要寫程式。

1

Check what you received 先確認收到的檔案

Unzip the folder I sent you. Inside you should see a file called index.html — this is the homepage. Other files (images, CSS, JS) sit next to it.

把我傳給你的壓縮檔解開。資料夾裡會看到一個 index.html，那就是首頁。其他檔案（圖片、樣式、JS）會跟它放在一起。

✓ Quick test · 先測試

Double-click index.html. Your browser should open it and show the website. If it works locally, it will work on GitHub.

直接點兩下 index.html，瀏覽器會自動打開你的網站。本機看得到，上 GitHub 就一定看得到。

2

Sign up for GitHub 註冊 GitHub 帳號

If you don't have an account yet, go to github.com and click Sign up. It's free. Pick a username you don't mind appearing in your website's URL.

如果還沒有帳號，到 github.com 點 Sign up 註冊，完全免費。挑一個你不介意出現在網址裡的帳號名稱。

Note · 提醒

Your username becomes part of your site address, like https://yourname.github.io/...

你的帳號會出現在網址裡，例如 https://yourname.github.io/...

3

Create a new repository 建立一個新的 Repository

After logging in, click the green "New" button (or the "+" at the top right → New repository).

登入後，點綠色的 "New" 按鈕（或右上角 "+" → New repository）。

  • Repository name: pick a short name (e.g. my-website). This appears in your URL.
  • 名稱挑短一點，例如 my-website，會直接出現在網址。
  • Public — must be Public for free GitHub Pages.
  • 必須選 Public（公開），免費版 Pages 限定。
  • Leave the other options as default. Click "Create repository".
  • 其他選項都保留預設，按下 "Create repository"。
4

Upload your folder 上傳你的資料夾

On your new empty repository page, look for the link that says "uploading an existing file", or click Add file → Upload files.

在剛建好的空 repo 頁面，找到 "uploading an existing file" 的連結，或點 Add file → Upload files。

  • Open your website folder on your computer.
  • 打開你電腦裡的網站資料夾。
  • Select everything inside (including index.html) and drag it into the browser upload area.
  • 把資料夾裡面的所有檔案（包含 index.html）全部拉進瀏覽器的上傳區。
  • Wait for all files to finish uploading. Scroll down and click "Commit changes".
  • 等檔案全部上傳完，往下捲，按 "Commit changes"。
Important · 注意

Drag the contents of the folder, not the folder itself. index.html must end up at the top level of the repository.

是拉資料夾「裡面」的檔案，不是整個資料夾。index.html 必須在 repository 的最外層。

5

Turn on GitHub Pages 開啟 GitHub Pages

In your repository, click Settings (top menu) → Pages (left sidebar).

在 repository 頁面，點 Settings（上方選單）→ Pages（左側選單）。

  • Under "Build and deployment", find Source and choose "Deploy from a branch".
  • 在 "Build and deployment" 區塊，Source 選 "Deploy from a branch"。
  • Under Branch, select main and folder / (root). Click Save.
  • Branch 選 main，資料夾選 / (root)，按 Save。
6

Wait a minute, then visit your site 等一分鐘，打開你的網站

GitHub takes about 1–2 minutes to publish. Refresh the Pages settings page — you'll see a green box with your live URL:

GitHub 大約需要 1–2 分鐘把網站發布上線。重新整理 Pages 設定頁，會看到一個綠色框，裡面就是你的網址：

https://<your-username>.github.io/<repository-name>/
★ Done · 完成

Open that URL in any browser. Share it with anyone — it works on phones, tablets, and computers worldwide.

用任何瀏覽器打開這個網址，就可以分享給任何人，手機、平板、電腦全世界都看得到。

Common Questions 常見問題

I want to update my website. How? 想更新網站內容怎麼辦？

Go to your repository, click the file you want to edit (e.g. index.html), then the pencil icon. Or upload a new version using the same drag-and-drop method. Changes appear online in about a minute.

回到 repository，點要修改的檔案（例如 index.html），按鉛筆圖示直接編輯；或用一樣的拖拉方式上傳新版本。改動約一分鐘後就會生效。

Can I use my own domain (like mysite.com)? 可以用自己的網域嗎（例如 mysite.com）？

Yes. In Settings → Pages, fill in the "Custom domain" field, and add the right DNS records at your domain provider. Ask me when you're ready.

可以。在 Settings → Pages 填入 "Custom domain"，並到網域商設定 DNS。到時候再問我就好。

My site shows "404" or appears broken. What do I check? 網站出現 "404" 或畫面壞掉？

Most often: (1) index.html isn't at the top level of the repository, or (2) image/CSS paths use uppercase/lowercase incorrectly. GitHub is case-sensitive — Photo.jpg and photo.jpg are different files.

最常見原因：(1) index.html 沒放在 repository 最外層；(2) 圖片或 CSS 路徑大小寫不對。GitHub 區分大小寫，Photo.jpg 跟 photo.jpg 是不同檔案。

Is this really free? Forever? 真的完全免費？永久嗎？

Yes — for public repositories with reasonable traffic (100 GB/month bandwidth limit, which is plenty for personal sites). No credit card required.

是的。公開 repository 的個人網站完全免費（每月 100 GB 流量上限，一般個人站根本用不完），也不需要綁信用卡。

What if I get stuck? 如果卡住了怎麼辦？

Take a screenshot of where you are and send it to me. Most issues are solved in one message.

截圖你卡住的畫面傳給我，多數問題一則訊息就能解決。
